- **青岛地铁 MMCC 外部接口 FEP 服务器**

```
1、项目名称：青岛地铁 MMCC 外部接口 FEP 服务器
2、项目描述：
    在最大程度不影响且尽量避免改变原有 SRS 服务器功能与性能的前提下，将 SRS 接收到的轨旁列车数据进行计算处理后转发至 MMCC（青岛地铁线网运营管理与指挥中心），将外部接口功能放到FEP实现跟SRS服务器的解耦合，方便后续在不影响SRS原有功能的情况下进行其他产品研发升级工作。
3、功能需求：
    a、ZC、CBI、OBCU 等轨旁列车设备向 SRS 服务器发送状态信息（既有）。
    b、FEP 服务器与 SRS 服务器进行网络通信，同步 SRS 处理的轨旁列车部分数据。
    c、FEP 向 MMCC 发送轨旁设备状态信息、列车位置信息、SRS 管理的列车计划信息、SRS 时刻表历史计划和行车数据。
    d、在尽可能不改动 SRS 的情况下，新增 FEP 服务器，并实现双机热备，当主机出现故障时，备机切换为主用服务器。
4、项目职责：
    依据过往经验，在项目前期完成 MMCC_FEP_ICD 接口文档的设计；运用 libevent 网络库构建网络通信模块，借助成熟稳定的网络框架迅速搭建起网络通信架构；增设无锁队列用于缓存网络数据包，优化网络数据包的处理流程，并解决 TCP 粘包和应用层协议分包的问题；通过 RPC 技术将实时变化的所需数据从 SRS 同步至 FEP 服务器，实现 FEP 与 SRS 功能的解耦合；应用观察者模式的注册通知机制，通过实现数据的实时更新完成事件驱动系统；采用 MySQL 存储列车历史时刻表数据，利用 Redis 缓存一月最新数据进行数据库优化，并结合数据库连接池提升连接速度。在双机热备的主备机之间添加应用层心跳包，用于检测服务器的存活状态。
5、技术栈：
    C++、RPC、protobuf、libevent、RingBuffer、Redis、MySQL、数据库连接池、线程池、spdlog、CMake、git
6、项目成果：
    将外部接口所需数据通过 SRS 使用 RPC 技术实时传输到 FEP ，实现了外部接口功能跟 SRS 服务器的解耦合;
    成功完成了 FEP 外部接口服务器的设计与编码工作，使其得以在青岛地铁项目中成功落地应用。通过精心设计和优化，确保了服务器的高效稳定运行，显著提升了轨旁列车数据处理与传输的效率和可靠性，为青岛地铁的运营管理与指挥提供了有力的技术支持;
```
